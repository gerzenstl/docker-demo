En la actualidad:

Linux containers (LXC)?

“Tecnología de virtualización disponible en Linux que permite ejecutar a su vez multiples instacias de Sist. Op. aislados.”

Container como solución
Linux container

  • Espacio reservado para procesos.

  • Propia interfaz de red.

  • Puede ejecutar procesos como root.

  • Tiene su propio /sbin/init

Componentes de Docker

Docker containers

  • Similar a un directorio con todo lo necesario para ejecutar la(s) aplicacion(es)

  • Generados a partir de Docker Images

  • LXC

Docker Images

  • Son templates (o snapshots) de containers o Sist. Operativos.

Dockerfile

  • Script que automatiza la generacion de imagenes (Docker Images).

Partes de Docker (cliente-servidor)

Docker deamon

  • Recibe y procesa requets a traves de su API.

Docker client

  • Una CLI que se comunica con el deamon.

Docker Hub

  • Registro online de imagenes.

  • Es el Docker deamon quien se comunica con el Hub en caso de obtener una imagen.

  • Donde esta? https://hub.docker.com.

Docker

Instalación:

  • Atención: checkear pre-requisitos según que Sist. Op. y version tengan.

  • Para Ubuntu requiere que sea x64 bits y kernel +3.10

  • Windows o OSX? boot2docker: VM ligera (~24MB) donde puede correr Docker.

  • En Ubuntu:

  •      wget -qO- https://get.docker.com/ | sh

  •      sudo docker run hello-world

Docker

Trabajando con imagenes

  • Listar imagenes en Host: docker images

  • Obtener imagen desde Hub: docker pull {imageName}

  • Borrar imagen: docker rmi {imageName}

  • Crear imagen a partir de Dockerfile: docker build -t {imageName} .

Docker

Trabajando con Containers:

  • Ejecutar container en modo interactivo: docker run -i -t {containerID} /bin/bash

  • Ejecutar container que corra en background: docker run -d {containerID}

  • Ver container en detalle: docker inspect {containerID}

  • Generar nueva imagen desde container: docker commit {containerID} [name]:[tag]

  • Detener container: docker stop {containerID}

Docker

Ventajas:

  • Es mas eficiente en manejo de recursos.

  • Permite dividir aplicacion de gran tamao en microservices.

  • Posibilidad de mover aplicaciones y sus configs a traves de entornos (dev, qa, staging, ...).

  • Permite tener entornos muy similares al de Producción (en todo lo posible).


Desventajas:

  • Soporte nativo solo para Linux.

  • Potencialmente menos seguras que una VM.

  • Hay resistencia en usarlo en entornos Producción.

Docker vs Vagrant

  • Cada una tiene sus pro y contras.

  • Depende del escenario de tu proyecto.

  • Vagrant y Docker? Si, se puede implementar.

  • Ambos soportan Puppet o Chef.
